# Spaste 보안팀 설명서

## 제품 목적
Spaste는 기업 내부 PC에서 텍스트/이미지 복사 이력을 로컬에만 저장하는 클립보드 매니저다. Mac의 Maccy와 유사한 사용 경험을 Windows 환경에 제공하되, 오프라인 운영과 감사 추적을 우선한다.

## 보안 설계 원칙
- 인터넷 통신 없음: 애플리케이션 런타임에는 외부 API 호출이나 텔레메트리가 없다.
- 사용자별 분리 저장: 모든 런타임 데이터는 `%APPDATA%\Spaste` 아래에 저장된다.
- 관리자 권한 최소화: 애플리케이션 실행, 설정 변경, 자동 시작 등록은 사용자 권한으로 수행한다.
- 감사 로그 제공: `%APPDATA%\Spaste\logs\YYYYMMDD.log`에 주요 이벤트가 텍스트 로그로 기록된다.

## 저장 위치
- 설정: `%APPDATA%\Spaste\settings.json`
- 히스토리 DB: `%APPDATA%\Spaste\history.db`
- 이미지 캐시: `%APPDATA%\Spaste\images`
- 로그: `%APPDATA%\Spaste\logs`

## 기록되는 이벤트
- 애플리케이션 시작/종료
- 텍스트/이미지/이미지 파일 캡처
- 사용자 재복사
- 설정 변경
- 오류 발생 시 스택 정보

## 미기록/비전송 항목
- 외부 서버 전송 데이터 없음
- 브라우저 쿠키, 토큰, 계정 정보 수집 없음
- 파일 시스템 전체 스캔 없음
- 관리자 권한 상승 요구 없음

## 자동 시작
자동 시작은 `HKCU\Software\Microsoft\Windows\CurrentVersion\Run`에 현재 사용자 기준으로만 등록된다. 도메인 환경에서 사용자별 정책 배포와 충돌하지 않도록 HKLM은 사용하지 않는다.

## 제거 정책
- Inno Setup uninstall 지원
- 설치 제거 시 프로그램 파일 삭제
- 사용자 데이터 폴더는 운영 정책상 수동 검토 후 삭제 가능하도록 별도 보관 가능
- 현재 스크립트는 `userappdata\Spaste`를 함께 정리하도록 설정되어 있음

## 지원 포맷
- 텍스트: Unicode Text
- 리치 텍스트 제거: HTML/RTF 감지 후 일반 텍스트 재적용
- 이미지: Clipboard bitmap
- 이미지 파일: 탐색기 파일 드롭 중 PNG/JPG/JPEG/BMP/GIF/TIFF/WEBP

## 운영 시 확인 포인트
- 백신/보안 에이전트가 클립보드 API(`AddClipboardFormatListener`)를 차단하지 않는지 확인
- 전역 단축키 `Ctrl+Shift+V` 충돌 여부 확인
- 대용량 이미지 사용 시 `%APPDATA%\Spaste\images` 용량 모니터링
